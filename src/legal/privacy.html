Privacy Policy

Effective date: June 30, 2026

This Privacy Policy explains how Portals handles information when you use the extension.

1. Data We Store

Portals stores your extension configuration in Chrome Sync Storage under your browser profile. This includes:

  • project profiles;
  • environment names and domains;
  • UI preferences (such as open in new tab and side-by-side options);
  • active profile selection and configuration version.

2. How Data Is Used

Stored configuration is used only to provide extension functionality: detecting active projects, building target URLs, handling command-based environment switching, and rendering popup settings.

3. Permissions and Access

Portals requests permissions required for core behavior:

  • storage: save and load your configuration;
  • tabs / activeTab: read current tab URL and open/update tabs;
  • system.display: arrange windows in side-by-side mode;
  • host permissions (<all_urls>): allow environment matching and URL switching across sites.

4. Data Sharing

Portals does not sell your data and does not intentionally share your configuration data with third parties. Extension data remains within Chrome extension storage mechanisms.

5. Remote Collection and Analytics

Based on the current code in this project, Portals does not include built-in analytics, ad trackers, or external telemetry endpoints for collecting personal data.

6. Security Considerations

While reasonable care is taken in implementation, no client-side software can guarantee absolute security. Avoid storing sensitive secrets in environment labels or domain fields.

7. Your Control

You can edit or delete stored profile data through the extension settings UI. You can also remove all extension data by uninstalling the extension from Chrome.

8. Changes to This Policy

This Privacy Policy may be updated as the extension evolves. Continued use after updates indicates acceptance of the revised policy.
